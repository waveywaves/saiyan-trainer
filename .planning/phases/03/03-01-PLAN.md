<objective>
Build a Docker image that packages BizHawk 2.11 with Xvfb, x11vnc, noVNC, and s6-overlay so that NEAT training runs headlessly inside a container. The container reads ROMs and save states from mounted volumes, writes genome checkpoints to a mounted output directory, and exposes the BizHawk display via noVNC web UI for live observation from a browser.

Purpose: Bridge between local BizHawk development (Phases 1-2) and Kubernetes orchestration (Phase 4). The container is the unit of deployment for Tekton Tasks.
Output: `docker/` directory with Dockerfile, s6-overlay service definitions, and a speed validation script. A buildable and runnable container image.
</objective>

  <action>
    Create the complete `docker/` directory tree with all files needed to build the BizHawk container image.

    **Dockerfile** (`docker/Dockerfile`):
    - Base image: `debian:bookworm-slim`
    - ARGs: `S6_OVERLAY_VERSION=3.1.6.2`, `BIZHAWK_VERSION=2.11`
    - Install system deps via apt: `mono-complete xvfb x11vnc fluxbox libopenal-dev lua5.4 liblua5.4-dev lsb-release mesa-utils libgtk2.0-0 libsdl2-2.0-0 wget unzip git python3 xz-utils ca-certificates`. Use `--no-install-recommends`. Clean apt cache after.
    - Install s6-overlay: ADD the noarch and x86_64 tarballs from GitHub releases, extract to `/`.
    - Install noVNC: `git clone --depth 1` noVNC and websockify repos into `/usr/share/novnc/` and `/usr/share/novnc/utils/websockify/`. Symlink `vnc.html` to `index.html`.
    - Download BizHawk: wget the `BizHawk-${BIZHAWK_VERSION}-linux-x64.tar.gz` release from GitHub into `/opt/bizhawk/`, strip one component, chmod +x `EmuHawkMono.sh`.
    - Create data directories: `/data/roms`, `/data/savestates`, `/data/output`, `/data/lua`.
    - Copy s6 service definitions: `COPY rootfs/ /`
    - Copy validation script: `COPY scripts/ /opt/scripts/`
    - Environment variables: `DISPLAY=:99`, `ENABLE_VNC=true` (default true per user requirement -- noVNC always running), `VNC_PORT=5900`, `NOVNC_PORT=6080`, `XVFB_RESOLUTION=1024x768x24`.
    - EXPOSE 6080.
    - ENTRYPOINT `["/init"]` (s6-overlay entrypoint).

    **s6-overlay services** -- each service needs a `type` file (containing `longrun`), a `run` script, dependency files, and a registration in the `user` bundle:

    1. **xvfb** (`rootfs/etc/s6-overlay/s6-rc.d/xvfb/`):
       - `type`: `longrun`
       - `run`: bash script that executes `Xvfb :99 -screen 0 ${XVFB_RESOLUTION} -ac +extension GLX +render -noreset`
       - No dependencies (root service).

    2. **fluxbox** (`rootfs/etc/s6-overlay/s6-rc.d/fluxbox/`):
       - `type`: `longrun`
       - `run`: bash script that sets `DISPLAY=:99` and executes `fluxbox`
       - `dependencies.d/xvfb`: empty file (depends on xvfb)

    3. **x11vnc** (`rootfs/etc/s6-overlay/s6-rc.d/x11vnc/`):
       - `type`: `longrun`
       - `run`: bash script that checks `ENABLE_VNC`. If not "true", log "VNC disabled" and `exec sleep infinity`. Otherwise `exec x11vnc -display :99 -forever -shared -rfbport ${VNC_PORT} -nopw -noxdamage`.
       - `dependencies.d/xvfb`: empty file

    4. **novnc** (`rootfs/etc/s6-overlay/s6-rc.d/novnc/`):
       - `type`: `longrun`
       - `run`: bash script that checks `ENABLE_VNC`. If not "true", log "noVNC disabled" and `exec sleep infinity`. Otherwise `exec /usr/share/novnc/utils/novnc_proxy --vnc localhost:${VNC_PORT} --listen ${NOVNC_PORT}`.
       - `dependencies.d/x11vnc`: empty file

    5. **bizhawk** (`rootfs/etc/s6-overlay/s6-rc.d/bizhawk/`):
       - `type`: `longrun`
       - `run`: bash script that sets `DISPLAY=:99` and executes `/opt/bizhawk/EmuHawkMono.sh --chromeless --lua="${LUA_SCRIPT:-/data/lua/main.lua}" --load-state="${SAVE_STATE:-/data/savestates/fight-start.state}" "${ROM_PATH:-/data/roms/rom.gba}"`.
       - `dependencies.d/xvfb`: empty file
       - `dependencies.d/fluxbox`: empty file

    6. **user bundle** (`rootfs/etc/s6-overlay/s6-rc.d/user/contents.d/`):
       - Create empty files named `xvfb`, `fluxbox`, `x11vnc`, `novnc`, `bizhawk` to register all services with s6-overlay's default bundle.

    **All `run` scripts must be executable (chmod +x)**. Use `#!/bin/bash` for scripts needing env var expansion (all except potentially xvfb, but use bash everywhere for consistency).

    **Validation script** (`docker/scripts/validate-speed.sh`):
    - Bash script that creates a minimal Lua script in `/tmp/benchmark.lua` which calls `emu.frameadvance()` 1000 times, measures elapsed time with `os.clock()`, prints FPS, prints PASS if >100 fps or WARNING if slower, and calls `client.exit()`.
    - The bash wrapper runs BizHawk with `--chromeless --lua=/tmp/benchmark.lua` against the mounted ROM.
    - Make executable.

    **.dockerignore** (`docker/.dockerignore`):
    - Ignore `*.md`, `.git`, `__pycache__`, `*.pyc`.

    **IMPORTANT anti-patterns to avoid (from RESEARCH.md):**
    - Do NOT bake ROMs into the image (legal issue). ROMs are always mounted at runtime.
    - Do NOT skip fluxbox -- BizHawk may not render correctly in Xvfb without a window manager.
    - Do NOT use a resolution higher than 1024x768x24 for Xvfb (wastes CPU/memory).
    - Always use absolute paths for `--lua` flag (BizHawk changes cwd to its install dir).
    - The `--chromeless` flag does NOT make BizHawk headless -- Xvfb is still required.
