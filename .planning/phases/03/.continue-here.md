<decisions_made>

- Build mGBA from source (0.10.5 tag) instead of using Debian package, because Debian only has mgba-sdl (no --script flag) and no mgba-qt package for arm64
- Use Qt frontend (mgba-qt) instead of SDL (mgba-sdl) because only Qt has --script CLI flag for automatic script loading (confirmed via GitHub issue #2981)
- Multi-stage Docker build: builder stage with build-essential+cmake+Qt dev libs, runtime stage with just Qt runtime libs
- Install to /opt/mgba with PATH update instead of system-wide install
- Visualization files (network_display, hud, species_timeline) stubbed to no-ops since mGBA lacks BizHawk-style gui.* overlay API. Stats logged to console instead
- Controller uses KEY_MAP table with bitwise OR for bitmask construction instead of C.GBA_KEY constants (those require mGBA's C constant binding which may not be available in all contexts)
- Signed integer reads use manual two's complement: val >= 0x8000 -> val - 0x10000

</decisions_made>
