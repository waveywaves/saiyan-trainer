**Part A: Dockerfile FreeType fix**

In `docker/Dockerfile`, add `libfreetype-dev` to the builder stage apt-get (line 19, alongside existing build deps) and `libfreetype6` to the runtime stage apt-get (line 53, alongside existing runtime deps). This enables mGBA's `USE_FREETYPE=ON` default so `painter:drawText()` works.

**Part B: Rewrite network_display.lua**

Completely rewrite `lua/vis/network_display.lua` to implement a MarI/O-style neural network visualization using the mGBA Painter API. The module must:

1. **Lazy-initialize a canvas overlay** (132px wide x 160px tall, positioned at top-left (0,0)). Follow the same pattern as `hud.lua` and `mem_overlay.lua`:
   - `canvas:newLayer(132, 160)` to create the layer
   - `image.newPainter(layer.image)` to get a painter
   - Try loading fonts from `/opt/mgba/share/mgba/scripts/demos/SourceSans3-Regular.otf` then `/usr/share/fonts/truetype/dejavu/DejaVuSansMono.ttf`
   - If no font available, load `pixel_draw.lua` as fallback text renderer
   - Store state in a module-level table, only init once

2. **Dual-renderer architecture**: Use Painter API for shapes (drawRectangle, drawLine, drawCircle) always. For text, use `painter:drawText()` if FreeType font loaded, otherwise use `PD.drawText(layer.image, ...)` from pixel_draw.lua.

3. **Layout** (adapted from MarI/O displayGenome):
   - Input column: x=10, 9 nodes (8 game state + bias) spaced vertically from y=25 to y=145 (~13px apart)
   - Output column: x=115, 8 nodes (GBA buttons) spaced vertically from y=25 to y=130 (~15px apart)
   - Hidden nodes: spring-layout positioned in x=35..95 range. Use 4 iterations of force-directed positioning from MarI/O: `c.x = 0.75*c.x + 0.25*other.x`, clamped to [35, 95]. Y position: hash the neuron ID to distribute vertically in [25, 145].
   - Node radius: 3px for all nodes

4. **Drawing**:
   - **Background**: Semi-transparent black rectangle (0xB0000000) covering 132x160
   - **Connections**: For each enabled gene, draw a line from source neuron position to destination neuron position. Color using the MarI/O weight convention: `connectionColor(weight, sourceNeuronActive)` where green=positive weight, red=negative weight, alpha based on whether source neuron has non-zero activation. Use PD.weightColor() from pixel_draw.lua for the color calculation.
   - **Nodes**: Draw filled circles at each neuron position. Fill color based on activation value using PD.nodeColor(). Draw border circle in contrasting color.
   - **Labels**:
     - Input labels (left-aligned at x=1): Use `Inputs.getInputLabels()` for names -- "P1HP", "P2HP", etc., plus "BIAS" for the 9th
     - Output labels (right side at x=118): Use `Controller.getButtonNames()` -- "A", "B", "L", "R", "Up", "Dn", "Lt", "Rt"
   - **Stats text** at top (y=2): "G:{gen} F:{fitness}" and "Sp:{species} Max:{maxFit}" on line 2 (y=10)

5. **Public API**:
   - `NetworkDisplay.displayGenome(genome, pool)` -- main draw function, takes genome and pool for stats
   - `NetworkDisplay.setEnabled(bool)` -- toggle overlay on/off
   - `NetworkDisplay.isAvailable()` -- returns true if canvas/painter initialized successfully
   - The module returns the NetworkDisplay table

6. **Performance**: The module does NOT throttle itself -- the caller (main.lua) is responsible for calling it every N frames. Each call clears and redraws the full overlay, then calls `layer:update()`.

7. **Graceful fallback**: If `canvas` global is not available (e.g., running outside mGBA), all functions become no-ops. Wrap init in pcall. Never crash the training loop.

Key details:
- Color format is 0xAARRGGBB (ARGB32)
- Use `painter:setFill(true)` + `painter:setFillColor()` for filled shapes
- Use `painter:setStrokeColor()` + `painter:setStrokeWidth(1)` for lines
- For drawLine: `painter:drawLine(x0, y0, x1, y1)`
- For filled circles: `painter:setFill(true)` then `painter:drawCircle(cx, cy, r)`
- Call `layer:update()` once at the end of each draw cycle
- Load Config, GameInputs, Controller via dofile at module level (same pattern as current network_display.lua)

Modify `lua/main.lua` to load and call the neural network overlay display during training.

1. **Load the module** near the top with the other module loads (after loading MemoryMap and Config):
   ```lua
   local NetworkDisplay = dofile("lua/vis/network_display.lua")
   ```

2. **Add a frame-throttled overlay draw** in the frame callback. Add a draw counter and call NetworkDisplay.displayGenome() every 5 frames. The genome to display is the currently-evaluating genome from the trainer's state:

   After the existing `callbacks:add("frame", function() ... end)` block, replace it with:
   ```lua
   local draw_counter = 0
   local DRAW_INTERVAL = 5  -- update overlay every 5 frames (~12 FPS visual update)

   callbacks:add("frame", function()
       trainer:tick()
       updateHUD()
       draw_counter = draw_counter + 1
       if draw_counter % DRAW_INTERVAL == 0 then
           -- Get current genome being evaluated
           local genome = nil
           if trainer.state == "evaluating" and trainer.pool then
               local sp = trainer.pool.species[trainer.speciesIdx]
               if sp then
                   genome = sp.genomes[trainer.genomeIdx]
               end
           end
           if genome then
               NetworkDisplay.displayGenome(genome, trainer.pool)
           end
       end
   end)
   ```

3. **Do NOT remove or modify the existing HUD** (console buffer). The network overlay and the console HUD serve different purposes and coexist independently. The network overlay draws on the game screen canvas; the console HUD writes to the scripting window text panel.

Key: The overlay only draws when `trainer.state == "evaluating"` because that's when a genome has an active network. During other states (waiting, init, gen_done), there's no genome to visualize.
