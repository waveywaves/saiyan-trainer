**lua/vis/network_display.lua**: Draws the evolved neural network topology on the BizHawk screen. Adapted from MarI/O's displayGenome function but using labeled fighting game inputs instead of a tile grid.

Structure:
1. Load dependencies:
   ```lua
   local Config = dofile("lua/neat/config.lua")
   local GameInputs = dofile("lua/game/inputs.lua")
   local Controller = dofile("lua/controller.lua")
   ```

2. Define layout constants:
   - Network display area: x=0, y=0, width=130, height=160 (left side of GBA screen 240x160)
   - Input nodes: arranged vertically on the left edge (x=10)
   - Output nodes: arranged vertically on the right edge of the display area (x=120)
   - Hidden nodes: positioned in the middle area using a simple force-directed layout

3. `displayGenome(genome)`: Main visualization function.
   a. If genome.network is nil, return (no network built yet)
   b. Draw semi-transparent background box for the network area: `gui.drawBox(0, 0, 132, 160, 0x80000000, 0x80000000)`
   c. Build cells table mapping neuron IDs to screen positions:
      - Input neurons (1..Config.Inputs): evenly spaced vertically on left. Label each with input name from GameInputs.getInputLabels() (plus "Bias" for the last one). Draw label text in gray (0xFF888888) using `gui.drawText(x, y, label, color, fontSize)` -- use fontSize 7 or 8 to fit.
      - Output neurons (Config.MaxNodes+1..Config.MaxNodes+Config.Outputs): evenly spaced vertically on right. Label each with button name from Controller.getButtonNames(). Color active outputs (value > 0) in green (0xFF00FF00), inactive in dark gray.
      - Hidden neurons: collect all neuron IDs from genome.genes that are not input/output. Position using MarI/O's iterative force-directed layout (4 iterations): push apart nodes that are too close, pull toward connected nodes. X position proportional to layer depth (inputs=left, outputs=right).
   d. Draw connections:
      - For each enabled gene in genome.genes:
        - Get source and target cell positions
        - Color: green (0xA000FF00) if weight > 0, red (0xA0FF0000) if weight < 0
        - Alpha/thickness proportional to |weight| (thin for small weights, thick for large)
        - `gui.drawLine(x1, y1, x2, y2, color)`
      - Skip disabled genes (or draw them as very faint dotted gray)
   e. Draw neuron nodes:
      - For each neuron in cells: draw a small box (4x4 pixels)
      - Brightness based on neuron activation value: `brightness = math.floor((value + 1) / 2 * 255)`. Clamp 0-255.
      - Fill color encodes brightness: brighter = more active
      - Border: black (0xFF000000)
      - `gui.drawBox(x-2, y-2, x+2, y+2, borderColor, fillColor)`

4. Helper functions:
   - `sigmoid01(x)`: maps [-1,1] to [0,1] for visualization brightness
   - `weightToColor(weight)`: returns ARGB color based on weight sign and magnitude
   - `positionHiddenNodes(genome, cells)`: force-directed positioning of hidden neurons

**lua/vis/hud.lua**: Training statistics heads-up display.
- `displayHUD(pool, currentGenome, fps)`: renders training stats on BizHawk screen.
  Position: bottom of screen or top-right corner (outside network display area).
  Display:
  - "Gen: {N}" -- current generation
  - "Species: {N}" -- count of species
  - "Genome: {N}/{M}" -- current genome being evaluated / total
  - "Fitness: {F}" -- current genome's fitness (or best so far)
  - "Max Fitness: {F}" -- all-time best
  - "Stale: {N}" -- staleness of current species
  Use `gui.drawText()` with white text on semi-transparent black background.
  Font size 8-9 to be readable but not intrusive.

- `displayFitnessBar(fitness, maxFitness)`: simple horizontal bar showing fitness relative to max.
  Green fill proportional to fitness/maxFitness.
  Position below HUD text.

Both modules return their table via module pattern.

**lua/vis/species_timeline.lua**: Tracks species membership over generations and renders a timeline visualization.

Structure:
1. Load config: `local Config = dofile("lua/neat/config.lua")`

2. Define `SpeciesTracker` module with internal state:
   - `history = {}` -- array of generation snapshots. Each snapshot: `{generation=N, species={}}`
   - Each species entry in snapshot: `{id=speciesIndex, count=genomeCount, topFitness=F, color=C}`
   - `maxGenerations = 50` -- how many generations to display (sliding window)
   - `colorPalette = {}` -- predefined colors for species (cycle through for new species)

3. Color palette: 12-16 distinct colors for species identification:
   ```lua
   {0xFFFF0000, 0xFF00FF00, 0xFF0000FF, 0xFFFFFF00, 0xFFFF00FF, 0xFF00FFFF,
    0xFFFF8800, 0xFF88FF00, 0xFF0088FF, 0xFFFF0088, 0xFF88FF88, 0xFFFF8888}
   ```
   Assign colors by species order of appearance. Reuse for extinct+reborn species.

4. `SpeciesTracker.record(pool)`:
   - Called once per generation after evaluation
   - Create snapshot of current species: for each species, record ID (index), genome count, topFitness
   - Assign consistent color to each species (track by species representative genome hash or just use index)
   - Append to history. If history > maxGenerations, shift oldest entry off.

5. `SpeciesTracker.display()`:
   - Renders species timeline as a stacked bar chart on BizHawk screen
   - Position: right side of screen, or use a separate overlay position (y=0, x=140, width=100, height=80)
   - Each column = one generation (width = displayWidth / min(#history, maxGenerations))
   - Each segment in column = one species, height proportional to genome count
   - Color = species color from palette
   - Draw generation numbers along bottom (every 10th generation)
   - Draw "Species Timeline" label at top
   - Use `gui.drawBox(x, y, x+w, y+h, color, color)` for filled rectangles

6. `SpeciesTracker.getSummary()`:
   - Returns text summary: "Gen {N}: {S} species, largest={L} ({P}%), smallest={S} ({P}%)"
   - Useful for console logging

7. `SpeciesTracker.getHistory()`:
   - Returns the full history table (for checkpoint saving if desired)

8. `SpeciesTracker.loadHistory(history)`:
   - Restores history from checkpoint data

Export via module pattern.

NOTE: BizHawk's gui.* API draws on top of the game screen each frame. The visualization is purely visual overlay -- it does not affect game state or training. All draw calls must happen between emu.frameadvance() calls to be visible.
