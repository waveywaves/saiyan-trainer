**lua/training/checkpoint.lua**: Full NEAT population serialization using dkjson.
- Load dkjson: `local json = dofile("lua/lib/dkjson.lua")`
- `saveCheckpoint(pool, filename)`:
  1. Build data table: `{generation=pool.generation, innovation=Innovation.getCurrent(), maxFitness=pool.maxFitness, species={}}`
  2. For each species: save topFitness, staleness, and all genomes
  3. For each genome: save maxneuron, fitness, mutationRates (all rate keys), and all genes
  4. For each gene: save into, out, weight, enabled, innovation (all 5 fields)
  5. Encode with `json.encode(data, {indent = true})` for human readability
  6. Write to file via `io.open(filename, "w")`
  7. Log: "Checkpoint saved: generation {N}, {M} species, fitness {F}"

- `loadCheckpoint(filename)`:
  1. Read entire file via `io.open(filename, "r") -> file:read("*all")`
  2. Decode with `json.decode(jsonStr)`
  3. Reconstruct pool table with proper structure (not just raw JSON -- ensure species/genomes/genes are proper tables with correct types)
  4. CRITICAL: Set `Innovation.reset(data.innovation)` to restore innovation counter (Pitfall 2)
  5. CRITICAL: Verify innovation counter > max innovation in any gene. If not, scan all genes and set innovation to max+1. Log warning if this correction is needed.
  6. Return reconstructed pool
  7. Log: "Checkpoint loaded: generation {N}, {M} species, {G} genomes"

- `getCheckpointFilename(generation)`: returns "checkpoints/gen_{generation}.json"
- `getLatestCheckpoint(dir)`: scans directory for gen_*.json files, returns the one with highest generation number (for auto-resume)

**lua/training/combo_logger.lua**: Input sequence recording and pattern analysis.
- `newLogger()`: returns `{log = {}, frameCount = 0}`
- `Logger:record(buttons)`: appends buttons table to log, increments frameCount
- `Logger:clear()`: resets log and frameCount
- `analyzeInputLog(inputLog)`:
  1. Convert each buttons table to a binary string key (one char per button, 1=pressed, 0=not). Use Controller.getButtonNames() order.
  2. Count trigram (3-frame) patterns using sliding window
  3. Sort patterns by frequency descending
  4. Calculate Shannon entropy over trigram distribution
  5. Return table:
     ```
     {
       topPatterns = array of {pattern=string, count=number, buttons=decoded_string},
       entropy = number,
       uniquePatterns = count,
       totalFrames = count,
       isButtonMashing = entropy < 1.5
     }
     ```
  6. Include helper to decode binary pattern string back to button names for readability: "10100010" -> "A+L+Down"

**lua/training/loop.lua**: The main training loop that drives NEAT evolution.

Load all dependencies via dofile:
```lua
local Config = dofile("lua/neat/config.lua")
local Pool = dofile("lua/neat/pool.lua")
local Network = dofile("lua/neat/network.lua")
local Innovation = dofile("lua/neat/innovation.lua")
local GameInputs = dofile("lua/game/inputs.lua")
local Fitness = dofile("lua/game/fitness.lua")
local Controller = dofile("lua/controller.lua")
local SaveState = dofile("lua/savestate_helper.lua")
local Checkpoint = dofile("lua/training/checkpoint.lua")
local ComboLogger = dofile("lua/training/combo_logger.lua")
```

Define module `TrainingLoop = {}`.

- `evaluateGenome(genome, pool)`:
  1. Load save state: `SaveState.resetFight()`
  2. Build network: `Network.generateNetwork(genome, Config)`
  3. Record starting health values via memory reads
  4. Initialize combo logger for this evaluation
  5. Initialize tracking vars: frameCount=0, lastDamageFrame=0
  6. Per-frame loop (`while true do`):
     a. Read game inputs: `local inputs = GameInputs.getGameInputs()`
     b. Forward pass: `local outputs = Network.evaluateNetwork(genome.network, inputs, Config)`
     c. Apply to controller: `Controller.applyController(outputs)`
     d. Record buttons in combo logger
     e. Track damage dealt (compare current P2 HP to start; update lastDamageFrame when damage increases)
     f. Check termination: round over (from memory), or frameCount > Config.TimeoutConstant
     g. If terminating: break
     h. Increment frameCount
     i. `emu.frameadvance()`
  7. Clear controller: `Controller.clearController()`
  8. Calculate fitness: call `Fitness.calculateFitness({...})` with all tracked values
  9. Set `genome.fitness = fitness`
  10. Store combo log on genome: `genome.comboAnalysis = ComboLogger.analyzeInputLog(logger.log)`
  11. IMPORTANT: All Lua tracking variables are reset at step 1-5 (Pitfall 5: save state does not reset Lua state)

- `evaluatePopulation(pool)`:
  1. For each species in pool.species:
     For each genome in species.genomes:
       If genome.fitness == 0 (not yet evaluated):
         `evaluateGenome(genome, pool)`
  2. Track and log: best fitness this generation, species count, generation number

- `runGeneration(pool)`:
  1. `evaluatePopulation(pool)`
  2. Update pool.maxFitness if new best found
  3. Log generation stats: "Gen {N}: best={F}, species={S}, genomes={G}"
  4. Save checkpoint every generation: `Checkpoint.saveCheckpoint(pool, Checkpoint.getCheckpointFilename(pool.generation))`
  5. Also save "latest" checkpoint: `Checkpoint.saveCheckpoint(pool, "checkpoints/latest.json")`
  6. Log combo analysis summary for the best genome
  7. Call `Pool.newGeneration(pool, Config, Innovation)` to breed next generation

- `runTraining(options)`: Main entry point.
  options table:
  ```
  {
    generations = 100,       -- how many generations to run (0 = infinite)
    resume = false,          -- attempt to load latest checkpoint
    checkpointFile = nil,    -- specific checkpoint to resume from
    opponents = nil,         -- table of save state files for multi-opponent rotation
    opponentRotation = 10,   -- rotate opponent every N generations
  }
  ```
  1. Create checkpoints/ directory if not exists
  2. If resume or checkpointFile: load checkpoint, restore pool and innovation counter
  3. Else: create new pool via `Pool.newPool(Config, Innovation)`
  4. Log training config: population size, inputs, outputs, target generations
  5. Generation loop: for gen = pool.generation, options.generations (or infinite):
     a. If opponents table provided: rotate save state file every opponentRotation generations (OPP-01, OPP-02). Swap the save state file that SaveState uses.
     b. Call `runGeneration(pool)`
     c. Log progress: "Generation {N}/{total} complete"
  6. Final save on completion or interrupt

Multi-opponent support (OPP-01, OPP-02):
- `setOpponentConfig(opponents)`: takes a table like:
  ```
  {
    {file = "savestates/vs_easy.State", desc = "Easy CPU"},
    {file = "savestates/vs_medium.State", desc = "Medium CPU"},
    {file = "savestates/vs_hard.State", desc = "Hard CPU"},
    {file = "savestates/vs_goku.State", desc = "vs Goku"},
    {file = "savestates/vs_vegeta.State", desc = "vs Vegeta"},
  }
  ```
- Rotation logic: `currentOpponent = opponents[(math.floor(pool.generation / opponentRotation) % #opponents) + 1]`
- Before each generation, if opponents configured, update the save state file used by resetFight

- Banner: print "=== Saiyan Trainer - NEAT Fighting Game AI ===" and version/date

- Call `TrainingLoop.runTraining(options)`
