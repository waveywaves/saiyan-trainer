Create `lua/controller.lua` -- the module that translates NEAT neural network outputs to GBA button presses.

Structure:
1. Define `ButtonNames` table (ordered array) with the 8 GBA buttons used for fighting:
   `{"A", "B", "L", "R", "Up", "Down", "Left", "Right"}`
   Note: Start and Select are excluded from NEAT control (not useful for fighting).

2. Define `NUM_OUTPUTS` constant = #ButtonNames (8). This is the number of output neurons the NEAT network needs.

3. Define `outputsToController(outputs)` function:
   - Takes an array of numbers (one per button, from NEAT output neurons)
   - Returns a table mapping button names to true/false
   - Threshold: `outputs[i] > 0` means pressed (NEAT uses sigmoid or tanh; positive = press)
   - This naturally supports simultaneous combos: if outputs[3] > 0 and outputs[6] > 0, both L and Down are pressed simultaneously
   - Example: outputs = {0.8, -0.2, 0.1, -0.5, -0.1, 0.9, -0.3, 0.7} produces {A=true, B=false, L=true, R=false, Up=false, Down=true, Left=false, Right=true}

4. Define `applyController(outputs)` function:
   - Calls `outputsToController(outputs)` then `joypad.set(result)`
   - Convenience wrapper for the training loop

5. Define `clearController()` function:
   - Sets all buttons to false: `joypad.set({})`
   - Used between evaluations to release all buttons

6. Add `getButtonNames()` getter and `getNumOutputs()` getter for NEAT network configuration.

7. Export via module pattern: `return Controller`

Create `lua/savestate_helper.lua` -- save state management for deterministic training:

Structure:
1. Define configurable `SAVE_STATE_DIR` = "savestates/"
2. Define `FIGHT_START_FILE` = SAVE_STATE_DIR .. "fight_start.State"
3. Define `resetFight()` function:
   - Calls `savestate.load(FIGHT_START_FILE, true)` (true suppresses OSD)
   - Logs to console: "Fight reset from save state"
   - Returns true if file exists and loads, false with error message if file missing
4. Define `createFightStartState()` function:
   - Calls `savestate.save(FIGHT_START_FILE, true)`
   - Logs: "Fight start save state created at: " .. FIGHT_START_FILE
   - Instructions comment: User should navigate to VS Mode fight, wait for countdown to finish, then call this function
5. Define `hasFightStartState()` function:
   - Checks if FIGHT_START_FILE exists using Lua io.open
   - Returns boolean
6. Register `event.onloadstate` callback that logs when any state is loaded (helps with debugging)
7. Export via module pattern

Create `lua/controller_test.lua` -- a BizHawk Lua script that demonstrates controller input by executing a predetermined sequence of moves, proving that button presses (including simultaneous combos) work correctly.

Structure:
1. Load controller module: `local controller = dofile("lua/controller.lua")`
2. Load savestate helper: `local ss = dofile("lua/savestate_helper.lua")`

3. Define a `MoveSequence` table -- an ordered list of moves to execute. Each move is a table with:
   - `name`: human-readable name (e.g., "Punch", "Special Move (Down+B)", "Dash Right")
   - `buttons`: table of button names to press (e.g., {"A"}, {"Down", "B"}, {"Right", "Right"})
   - `frames`: how many frames to hold the buttons (e.g., 5, 10, 15)
   - `pause_after`: frames of no input after the move (e.g., 10)

4. Define these test moves (DBZ fighting game common inputs):
   - "Idle" -- no buttons, 30 frames (baseline)
   - "Punch" -- {A=true}, 5 frames
   - "Kick" -- {B=true}, 5 frames
   - "Move Right" -- {Right=true}, 30 frames
   - "Move Left" -- {Left=true}, 30 frames
   - "Jump" -- {Up=true}, 10 frames
   - "Crouch" -- {Down=true}, 10 frames
   - "Special Move (Down+B)" -- {Down=true, B=true}, 10 frames (SIMULTANEOUS COMBO)
   - "Strong Attack (A+B)" -- {A=true, B=true}, 5 frames (SIMULTANEOUS COMBO)
   - "Dash Attack (Right+A)" -- {Right=true, A=true}, 10 frames (SIMULTANEOUS COMBO)
   - "Block (L shoulder)" -- {L=true}, 20 frames
   - "Ki Blast (R shoulder)" -- {R=true}, 10 frames
   - "Full Combo (Down+Right+B)" -- {Down=true, Right=true, B=true}, 10 frames (3-BUTTON COMBO)

5. Main execution loop:
   - Print startup banner: "=== Saiyan Trainer Controller Test ==="
   - Print: "Executing {N} test moves..."
   - Optionally reset fight if save state exists: `if ss.hasFightStartState() then ss.resetFight() end`
   - For each move in MoveSequence:
     a. Log to console: "Move: {name} [{buttons}] for {frames} frames"
     b. Display move name on screen via gui.text(10, 10, "Move: " .. name)
     c. For each frame in the move duration: set joypad with the button table, call emu.frameadvance()
     d. Clear controller for pause_after frames (neutral input between moves)
   - After all moves: print "Controller test complete. All {N} moves executed."
   - Final summary: print how many single-button and multi-button (combo) moves were tested

6. The script should be runnable standalone in BizHawk: `EmuHawk --lua=lua/controller_test.lua rom.gba`

NOTE: This script does NOT use the outputsToController function (that's for NEAT). It directly uses joypad.set with button tables to prove hardware-level input works. It tests CTRL-02 (simultaneous combos) explicitly with Down+B, A+B, Right+A, and Down+Right+B moves.
